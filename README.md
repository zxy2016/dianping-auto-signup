# 大众点评霸王餐自动报名脚本

基于 pyautogui + OpenCV 图像识别的 macOS 镜像模式自动化脚本。

## 功能特点

- 🎯 自动浏览霸王餐商品列表
- 🖱️ 自动点击"免费抽"、"我要报名"、"确认报名"
- 📱 自动滑动加载更多商品
- ✅ 自动验证报名成功并返回列表
- 📊 统计报名结果
-  拟人化操作，降低风控风险

## 环境要求

- macOS 12.0+
- Python 3.9+
- iPhone 镜像模式（已开启辅助功能权限）
- 大众点评 APP（已登录账号）

## 权限要求（重要！）

运行脚本前，需要在 macOS 系统设置中给**终端/IDE**开通以下权限：

| 权限 | 用途 | 设置路径 |
|------|------|----------|
| **屏幕录制** | 截取屏幕进行图像识别 | 系统设置 → 隐私与安全性 → 屏幕录制 |
| **辅助功能** | 控制鼠标点击和滚动 | 系统设置 → 隐私与安全性 → 辅助功能 |

**注意**：开通权限后需要**重启终端/IDE**才能生效。

## 快速开始

### 1. 切换到项目目录

```bash
cd dazhongdianp-auto-signup
```

### 2. 激活虚拟环境

```bash
source venv/bin/activate
```

### 3. 安装依赖（如需要）

```bash
pip install -r requirements.txt
```

### 4. 采集图像模板（重要！）

**首次运行必须先采集模板！**

按照 `templates/README.md` 的说明，手动操作一次并截取以下 UI 元素：

| 文件名 | 说明 |
|--------|------|
| `free_lottery.png` | "免费抽"按钮 |
| `signup_btn.png` | "我要报名"按钮 |
| `confirm_btn.png` | "确认报名"按钮 |
| `view_more.png` | "查看更多活动"按钮 |
| `success_hint.png` | 报名成功提示（可选） |

截图保存到 `templates/` 目录。

### 5. 启动手机镜像并进入页面

1. 打开 Mac 的"iPhone 镜像"应用
2. 打开大众点评 APP
3. 手动进入"霸王餐"模块

### 6. 运行脚本

```bash
# 报名 7 个商品
python main.py 7

# 不限制数量（遍历所有）
python main.py
```

## 命令行参数

| 命令 | 说明 |
|------|------|
| `python main.py` | 不限制数量，遍历所有商品 |
| `python main.py 5` | 报名 5 个商品 |
| `python main.py 10` | 报名 10 个商品 |

## 配置说明

编辑 `config.py` 调整参数：

```python
# 点击等待时间
CLICK_WAIT_TIME = 1.5            # 点击后等待
SUCCESS_WAIT_TIME = 3.0          # 报名成功等待

# 图像识别
IMAGE_RECOGNITION_THRESHOLD = 0.8  # 识别阈值
MAX_SWIPE_COUNT = 5                # 最大滑动次数
MAX_ITEMS_PER_PAGE = 5             # 每页最大商品数

# 日志配置
LOG_LEVEL = "INFO"
```

```
1. 检查模板文件 → 2. 验证在霸王餐页面 → 3. 循环处理商品
   ↓
```

## 故障排查

### 问题：找不到模板文件

**解决**：确保截图已保存到 `templates/` 目录，文件名正确。

### 问题：无法点击按钮

**可能原因**：
1. 模板截图尺寸与实际不符 → 重新截取更精确的截图
2. 屏幕缩放问题 → 确保截图时与实际显示尺寸一致
3. 按钮位置变化 → 调整识别阈值 `IMAGE_RECOGNITION_THRESHOLD`

### 问题：点击"免费抽"后没有进入详情页

**解决**：
- 这是 iPhone 镜像应用的特性，第一个点击可能不生效
- 脚本已自动处理：第一个商品的"免费抽"会执行双击

### 问题：滑动后没有新内容

**解决**：
- 检查网络连接
- 增加滑动距离（修改 `_swipe_page` 方法中的 `scroll_amount`）
- 减少 `MAX_SWIPE_COUNT` 避免无效滑动

### 问题：Mac 镜像模式无法连接

**解决**：
- 确保 macOS 辅助功能权限已开启
- 重启镜像模式
- 检查设备连接状态

### 问题：提示"could not create image from display"

**解决**：
- 检查是否已授予终端/IDE"屏幕录制"权限
- 重启终端/IDE 使权限生效
- 确保 iPhone 镜像窗口可见，未被其他窗口遮挡

### 问题：脚本运行几秒后提示超时

**说明**：每个操作步骤（查找按钮、点击按钮）有 30 秒超时限制
- 如果 30 秒内找不到按钮，会判定为超时
- 模板匹配置信度低于 0.6 会导致找不到按钮 → 重新采集模板

## 注意事项

⚠️ **重要提示**：

1. **合理使用**：请勿短时间内大量报名，避免被系统识别为异常行为
2. **账号安全**：建议设置合理的报名数量上限
3. **网络环境**：确保网络稳定，避免报名中途断网
4. **屏幕尺寸**：不同设备可能需要重新采集模板
5. **权限要求**：Mac 镜像模式需要辅助功能权限和屏幕录制权限
6. **第一个商品双击**：由于 iPhone 镜像特性，第一个商品的"免费抽"按钮需要双击才能生效，脚本已自动处理
7. **超时机制**：每个操作步骤有 30 秒超时，超时后自动结束

## 技术原理

- **pyautogui**：模拟鼠标点击、滚动操作
- **OpenCV**：模板匹配识别 UI 元素
- **Retina 坐标转换**：自动处理 macOS 屏幕缩放
- **拟人化操作**：贝塞尔曲线移动、随机延迟、随机滚动
